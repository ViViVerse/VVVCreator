You have successfully created a new ViViVerse organelle module!

A project folder with all required files has been created. This is what you have to do now:

Windows (MSBuild)

  1. Add the MSBuild project to the package solution (Organelle/Module).

  2. For organelles from the same package:
    1. Add references to the organelle collection and the symbiosis library to the project. If other common libraries are needed, add them too.
    2. Add references to the information structures of the organelles you want to have in the module to the .cpp file. The .cpp file comes with an example.
    3. If the organelles have faces, add the related .res files to the link command. The .res files can be found in the software/library folder and have the same names as the organelles with vvv_ in front.

  3. If you want to use organelles from other packages:
    1. Add the organelle library and symbiosis library headers of the package to the pch.h file. Use paths relative to the software/package folder. If other common libraries of the package are needed, add them as well.
    2. To the .cpp file, add references to the information structures of the organelles you want to have in the module. The .cpp file comes with an example.
    3. Add the collection and symbiosis libraries of the organelles to the link command.
    4. If the organelles have faces, add the related .res files to the link command. The .res files can be found in the software/library folder and have the same names as the organelles with vvv_ in front.

Linux, Android, Windows (CMake)

  1. Add the module folder to the CMakeLists.txt file of the package.

  2. For organelles from the same package:
    1. To the .cpp file, add references to the information structures of the organelles you want to have in the module. The .cpp file comes with an example.
    2. Windows only: if the organelles have faces, add the related .res files to the link command. The .res files can be found in the software/library folder and have the same names as the organelles with vvv_ in front.

  3. If you want to use organelles from other packages:
    1. Add the organelle library and symbiosis library headers of the package to the pch.h file. Use paths relative to the software/package folder. If other common libraries of the package are needed, add them as well.
    2. Add references to the information structures of the organelles you want to have in the module to the .cpp file. The .cpp file comes with an example.
    3. Add the collection and symbiosis libraries of the organelles to the link command.
    4. Windows only: if the organelles have faces, add the related .res files to the link command. The .res files can be found in the software/library folder and have the same names as the organelles with vvv_ in front.
